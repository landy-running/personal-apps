# Wanoku JMA Tide Prediction Admin Route

Phase 3D-2B adds a manual admin route for ingesting one official JMA tide prediction text file into Hydro-Coastal Persistence.

## Route

```text
POST /admin/collect-jma-tide-prediction
```

Authentication reuses the existing Worker admin mechanism:

- `WANOKU_ADMIN_SECRET`
- `Authorization: Bearer <secret>`
- or `X-Wanoku-Admin-Secret: <secret>`

The route is not connected to Cron or public PWA UI.

## Request body

```json
{
  "stationId": "TK",
  "sourceYear": 2026,
  "forecastIssuedAt": "2025-12-30T00:00:00.000Z"
}
```

Validation:

- `Content-Type` must be `application/json`.
- Body must be a small JSON object.
- `stationId`, `sourceYear`, and `forecastIssuedAt` are required.
- `forecastIssuedAt` must be canonical UTC ISO.
- Unsupported fields such as `sourceUrl`, `sourceName`, and `attribution` are rejected.
- `sourceUrl` is never accepted from the request body.

## Official source catalog

The route fetches only URLs generated by the local catalog:

```text
https://www.data.jma.go.jp/kaiyou/data/db/tide/suisan/txt/{year}/{stationId}.txt
```

Currently supported:

- `sourceYear`: `2026`
- `stationId`: `TK`, `CB`, `KZ`, `QS`, `TT`

Station metadata comes from `JMA_TIDE_PREDICTION_STATIONS_2026`.

## Forecast issuance metadata

The JMA fixed-width line does not contain a reliable forecast issuance timestamp. Therefore:

- `forecastIssuedAt` is caller supplied.
- the route does not infer it from fetch time.
- the route does not infer it from HTTP `Date` or `Last-Modified`.
- the route does not generate it from current time.

## Response

The response includes structured ingestion and persistence diagnostics:

- `ok`
- `status`
- `stationId`
- `sourceYear`
- `sourceRunId`
- `sourceUrl`
- `requestedAt`
- `completedAt`
- `forecastIssuedAt`
- `httpStatus`
- `rawHash`
- `sourceByteLength`
- `parsedObservationCount`
- `parserErrorCount`
- `parserWarningCount`
- `persistence`
- `errors`
- `warnings`

The response does not include raw source text, raw bytes, `ArrayBuffer`, `Response`, stack traces, authorization headers, or admin secret values.

## HTTP status mapping

- `200`: `status=ok` and persistence succeeded.
- `207`: partial parser result with valid observations saved.
- `400`: request validation error.
- `403`: admin secret mismatch.
- `503`: admin secret or D1 binding not configured.
- `502`: upstream fetch, HTTP, decode, or parse failure.
- `500`: persistence or unexpected internal failure.

## Production-scale behavior

One common-year station file contains:

- 365 fixed-width lines
- 24 hourly observations per line
- 8,760 observations per station

The initial five stations would produce about 43,800 observations, but the route intentionally processes only one station per request.

Persistence uses JSON payload chunks and D1 `json_each(?)` bulk inserts. The 2026 TK annual fixture test inserted 8,760 observations in 17 repository statements, with payload chunks below `D1_JSON_PAYLOAD_MAX_BYTES = 1_500_000`.
